# Changelog

> バージョン形式を `YYYY-MM-DD-連番` に変更（以前は v0.1.x）

## 2026-03-08-11 - 2026-03-08

- ミュート中はスライダーをグレーアウト表示
- 音量・ミュート状態をセーブデータに保存・復元

## 2026-03-08-10 - 2026-03-08

### 修正
- 最終マップでボス以外のモンスターを倒してもゲームクリアになるバグを修正（`isFinal` に加えて `isBoss` も確認するよう変更）
- 一括討伐モードでも同様の条件を修正

## 2026-03-08-9 - 2026-03-08

### 変更
- ステータスをタブから外し、右カラム上部に常時表示の固定枠として分離
- タブは アイテム / ショップ / 精製 の3つに変更、デフォルトはアイテム

## 2026-03-08-8 - 2026-03-08

### 変更
- 右カラムをアコーディオン（開閉式）からタブ切り替えUIに変更
  - ステータス / アイテム / ショップ / 精製 の4タブ
  - タブボタンはアイコン＋ラベル表示、アクティブタブをハイライト
  - ショップのゴールド表示をショップタブ内ヘッダーへ移動

## 2026-03-08-7 - 2026-03-08

### 追加
- 精製システムを追加（`data/refine.js`）
  - 複数アイテムを消費して精製品を生成するパネルを追加
  - ×1 / ×10 / ∞（無限）の連続精製に対応
  - 精製には一定時間が必要（100ms ごとに進捗更新）、途中キャンセル時は素材を返却
  - 精製中状態はセーブされ、ロード後も再開
  - 精製品3種を `data/items.js` に追加：精製コア / 魔力精髄 / 竜晶
    - 竜晶は精製品（精製コア・魔力精髄）を素材として要求

## 2026-03-08-6 - 2026-03-08

### 追加・修正
- マップごとに枠外ページ背景を設定できる `pageBg` フィールドを `MAP_DEFS` に追加（マップ切替時に body 背景を更新）
- 草原マップの `bg`（枠内）を `pageBg`（枠外）へ移動
- `PAGE_BG` が設定されている場合はマップ別 `pageBg` を無視する優先制御を追加

## 2026-03-08-5 - 2026-03-08

### 追加
- ゲーム枠外（ページ背景）の画像スロット `PAGE_BG` を追加 — `game.js` の `PAGE_BG` に画像パスを設定するとbody背景に表示

## 2026-03-08-4 - 2026-03-08

### 修正
- `#stage-bg`・`#game-bg` が表示されなかった不具合を修正 — `#game` に `isolation: isolate` を追加してスタッキングコンテキストを確立し、負 z-index の背景要素が `#game` の背景色の裏に隠れる CSS 仕様の問題を解消
- `#game-bg` がゲーム背景色（`--game-bg`）のフォールバックを担うよう変更（`#game` の `background` を削除）

## 2026-03-08-3 - 2026-03-08

### 追加
- ゲーム枠の背景画像スロット（`GAME_BG`）を追加 — `game.js` の `GAME_BG` に画像パスを設定すると枠全体に表示（空文字で無効）

## 2026-03-08-2 - 2026-03-08

### 変更
- INT が命中率・クリティカル率にも補正するよう追加（命中 +INT×0.1%、クリ +INT×0.5%）

## 2026-03-08-1 - 2026-03-08

### パフォーマンス改善（長時間起動時の重さ対策）
- `computePlayerStats()` にキャッシュを追加 — 毎アタックtickで走っていた全MAP/全アイテム走査を、状態変化時のみ再計算するよう最適化（ゲーム進行につれ顕著に改善）
- `flashEnemyHit()` の強制レイアウト（`void offsetWidth`）を廃止 — アニメーション実行中は再トリガーをスキップする方式に変更
- `playDefeatSound()` でAudioノードを `onended` 時に明示的に `disconnect()` — 音声グラフへのノード蓄積を防止

## 2026-03-07-4

### 変更
- 一括討伐モードの戦闘を全面刷新（爽快感重視）
  - 必中・一撃必殺（命中/回避/クリティカル判定なし、HPに関わらず即撃破、`KILL` 表示）
  - 攻撃速度を通常の 3 倍（`attackInterval / 3`、最低 100ms）
  - 貫通ダメージを廃止（1 体ずつ確実に撃破）
  - 効果音の自動再生ポリシー対策：ユーザー操作時に AudioContext を resume、サスペンド中の音は破棄して溜め込まない

## 2026-03-07-3

### 追加
- 一括討伐モードにレアモンスター出現（各ラウンドに最大 1 体、ボス枠を除くスロットで確率判定）
- オーバーキル貫通：余剰ダメージが次の敵に引き継がれ、連続撃破が連鎖する（「貫通撃破！」ログ）

### 変更
- 一括討伐の表示を**カルーセル方式**に刷新
  - 全 10 体を横スクロール列に並べ、現在の攻撃対象が常に中央に表示される
  - 倒すと列が左へスライドし次の敵が中央へ（右 = 生存、左 = 撃破済み）
  - カードの枠・背景・HP バーを非表示にし、モンスター画像のみをシンプルに表示
  - 画像サイズを通常戦闘と同じ 110px に統一し、戦闘枠の大きさを合わせた
  - 現在ターゲットは scale(1.12) で拡大ハイライト
  - ボスが最後の 1 体になると画像に `drop-shadow` グロー演出（枠なし）
  - `scrollIntoView({ inline: "center" })` で確実に中央配置
- `updateMultiGrid()` が `updateMultiStats()` を呼ぶようにし、ステータス表示をリアルタイム更新
- 一括討伐中のステータス表示を現在ターゲットの個別 HP / ATK + 残り体数表示に変更
